# 머슬킵 출시 체크리스트 (순서대로)

앱 코어(W1~W6)는 완성됨. 아래는 "실제 유료 출시"까지 순서.
✅ = 완료 · ⬜ = 남음(대부분 Teo 계정·결제·서류 필요) · 🤖 = Claude가 대신 가능

## 0. 지금 바로 가능 (개발 무관, 제일 중요)
- ⬜ 네이버 다이어트 카페 1곳 + 스레드에서 **Day 1 콘텐츠 게시** 시작 (`glp1-muscle-check/docs/community-content.md`)
- ⬜ hyojaitem 틱톡에 자가진단 계산기 링크 노출 (아래 유통 전략)

## 1. 법무·사업 (유료 결제 전 필수)
- ⬜ **사업자등록** (홈택스, 무료)
- ⬜ **통신판매업 신고** (정부24)
- 🤖✅ 이용약관 초안(`terms.html`) — ⬜ 사업자정보 채우고 법률 검토
- 🤖✅ 개인정보처리방침 초안(`privacy.html`) — ⬜ 위탁사(PG/호스팅) 확정 후 갱신
- 🤖✅ "의료 아님" 면책 — 앱·문서에 반영됨

## 2. 결제 (수익화)
- ⬜ **토스페이먼츠**(또는 포트원) 가입 + 사업자 심사
- 🤖 `app.js`의 `activateSub()` / `subPay` 스텁에 실제 결제창 연동
- ⬜ 정기결제(빌링) 설정 + 환불 규정 확정
- ⬜ 대금은 **본인 명의 직접 정산**(자금 미보유 → PG 등록 리스크 회피)

## 3. AI 추천 고도화 (선택 — 규칙기반으로도 출시 가능)
- ⬜ 서버리스 함수 호스트(Vercel Functions/Cloudflare Workers, 무료 티어) + LLM API 키
- 🤖 `aiRecommend()` 훅에 연결 → "오늘 뭐먹지"를 LLM 개인화로 업그레이드

## 4. 브랜드·배포
- ⬜ **도메인 musclekeep.app** 구매 ($9.99/년)
- 🤖 도메인 → GitHub Pages 연결(커스텀 도메인 + DNS)
- 🤖 PNG 아이콘(192/512) 생성(현재 SVG)
- 🤖 대기자 수집 실동작(`glp1-muscle-check`의 Formspree 연결)

## 5. 출시 후 지표(트리거)
- 무료 계산기 주 100명 유입 → SEO/콘텐츠 증량
- 앱 대기자/체험 30명 → 피드백 반영
- 유료 30명(월 30만원) → 채널 검증 / 100명(월 100만원) → 연령·글로벌 확장

## 우선순위 한 줄
**"1(법무·사업) + 2(결제) + 4(도메인)"만 하면 유료 출시 가능.** 3(LLM)은 규칙기반으로 대체되어 있어 나중에 해도 됨. 그리고 이 모든 것보다 **0(유저 획득)**이 성패를 가름.
